# Build

## Overview
Implement a task from the backlog using Development Agent with proper git workflow.

## Steps

1. **Pre-Implementation Check**
   - Display current git branch
   - Suggest feature/fix branch creation if needed
   - Read task details and acceptance criteria
   - Identify 2-3 highest-risk aspects

2. **Implementation**
   - Write production-quality code following project conventions
   - Implement comprehensive error handling and logging
   - Follow DRY, SOLID principles
   - Consider edge cases and failure modes
   - Add comments only for complex logic

3. **Post-Implementation**
   - Update task status in task management tool
   - Verify all acceptance criteria are met
   - Check for security vulnerabilities
   - Ensure integration with existing systems

## Checklist
- [ ] Git branch created/checked
- [ ] Task details reviewed
- [ ] Code implemented with error handling
- [ ] All acceptance criteria met
- [ ] Task status updated

## Usage
```
/build #123 OAuth2 Google provider integration
```
